# openclaw-zentao-pack 模块与脚本映射清单

本文档用于梳理 openclaw-zentao-pack 中每个模块对应的脚本入口、脚本类别和主要职责，作为后续梳理权限点、角色边界和指令路由的基础文档。

## 1. 使用目的

这份清单主要用于解决 4 个问题：

1. 每个模块到底依赖哪些脚本
2. 每个脚本属于查询、动作、回调还是同步
3. 哪些脚本被多个模块复用
4. 后续如何把模块映射成权限点与角色能力

## 2. 脚本分类说明

当前技能包中的脚本大致分为以下几类：

### 2.1 查询脚本

主要位于 scripts/queries/，用于读取禅道数据，不直接修改业务对象。

### 2.2 动作脚本

统一位于 scripts/actions/，用于创建、更新、关联、评审、执行等操作。根级 scripts/ 仅保留少量非重复入口脚本，例如产品初始化、任务聚合查询与用户同步。

### 2.3 回调脚本

主要位于 scripts/callbacks/，用于处理企业微信回调。

### 2.4 回复脚本

主要位于 scripts/replies/，用于企业微信机器人消息回复。

### 2.5 共享脚本

主要位于 scripts/shared/，用于共享客户端与公共封装。

### 2.6 同步脚本

主要用于企业微信用户同步到禅道。

## 4. 未直接挂模块但属于技能包的重要脚本

### 4.3 查询辅助脚本

- scripts/queries/_query_utils.ts
- scripts/queries/_acceptance_utils.ts

### 4.4 动作目录镜像脚本

目录 scripts/actions/ 中存在一组与根级脚本名称相近的动作脚本，例如：

- scripts/actions/create_bug.ts
- scripts/actions/create_story.ts
- scripts/actions/create_task.ts
- scripts/actions/create_release.ts
- scripts/actions/update_task_status.ts
- scripts/actions/update_bug_status.ts
- scripts/actions/update_release_status.ts
- scripts/actions/update_story_status.ts
- scripts/actions/update_testtask_status.ts

这部分脚本需要后续再确认它们与根级入口脚本之间的关系，通常可能属于：

- 新的分层入口
- 动作实现层
- 历史重构遗留的平行结构

## 6. 对后续权限设计的价值

这份映射清单可直接支撑下一步工作：

### 6.1 模块 -> action 权限点

可以继续把每个模块对应的脚本动作抽象为权限点，例如：

- story.create
- story.review
- task.update
- bug.assign
- release.create
- user.sync

### 6.2 角色 -> 模块 / 脚本边界

可以继续把角色映射成允许访问的模块和脚本，例如：

- 产品经理可用：产品、需求、发布模块
- 开发可用：任务、Bug 修复模块
- 测试可用：测试用例、测试单、Bug 提交与回归模块
- 发布负责人可用：发布、上线、验收模块

### 6.3 机器人路由治理

后续也可以基于该文档，把机器人指令路由到明确模块，而不是直接路由到零散脚本。

## 7. 后续建议

建议继续新增两份文档：

## 8. 模块去重原则

- `task-close-flow` 后续向 `task-status-flow` 合并
- `bug-regression-flow` 后续向 `testing-bugflow` + `bug-status-flow` 合并
- `testtask-status-flow` 后续向测试单主链路合并
- `lifecycle-closure-flow` 后续向 `acceptance-closure` 合并
- 后续新增功能不得重复落到上述待收敛模块中
