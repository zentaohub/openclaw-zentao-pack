# 批量导入任务需求说明

本文档用于描述 `openclaw-zentao-pack` 中“通过 Excel 或地址批量创建禅道任务”的本地需求背景、能力范围和后续演进方向。

## 1. 背景

当前技能包已经支持单条创建任务，但在真实业务中，用户经常会先整理一份 Excel 任务拆解表，再希望一次性导入禅道。

典型场景包括：

- 产品或项目经理先在 Excel 中拆分一批研发任务
- 测试或实施人员提供一批待处理事项
- 用户通过企业微信发送文件或链接，希望机器人自动创建对应任务

为了减少重复录入，需要在本地先补齐一条稳定的“批量导入任务”链路。

## 2. 目标

本阶段目标如下：

- 支持通过本地文件导入 `.xlsx` / `.csv`
- 支持通过 URL 下载 `.xlsx` / `.csv` 后导入
- 将表格中的每一行转换为禅道任务并创建到指定 `execution`
- 在企业微信文本入口中支持“导入任务 + URL + 执行ID”的触发方式
- 返回成功数、失败数和失败明细，便于回查

## 3. 非目标

本阶段不包含以下内容：

- 不直接修改服务器环境
- 不接入企业微信 `media_id` 文件下载链路
- 不自动推断执行、模块、需求的业务归属
- 不处理复杂审批、二次确认或重复导入去重策略

## 4. 输入方式

### 4.1 命令行导入

支持命令示例：

```bash
npm run import-tasks-from-excel -- --source-file examples/task-import-template.csv --execution 12 --userid admin --dry-run
```

```bash
npm run import-tasks-from-excel -- --source-url "https://example.com/tasks.xlsx" --execution 12 --userid admin
```

### 4.2 企业微信文本触发

支持示例：

```text
导入任务 https://example.com/tasks.xlsx 执行 12
```

如果缺少 URL 或 `execution`，机器人应返回最小必要缺参提示。

## 5. 表格字段约定

建议表头如下：

- `任务名称`
- `指派给`
- `预计工时`
- `优先级`
- `描述`
- `开始日期`
- `截止日期`
- `模块ID`
- `需求ID`

其中：

- `任务名称` 为必填
- `execution` 不从表格读取，而由命令参数或企微消息显式提供
- `指派给` 缺省时可回退到消息用户或默认账号

## 6. 处理流程

当前本地实现流程如下：

1. 接收 `source-file` 或 `source-url`
2. 读取或下载导入文件
3. 解析第一张工作表或 CSV 内容
4. 做表头映射和行级字段转换
5. 为每一行补齐统一的 `execution`
6. 调用现有 `createTask` 能力逐行创建
7. 汇总成功、失败和失败原因
8. 返回结构化 JSON 与简要 `reply_text`

- `README.md`

## 8. 当前限制

当前限制包括：

- 任务必须挂到指定 `execution`，不能只传产品 ID
- 企业微信目前只支持文本中的 URL 入口，不支持直接上传附件后自动下载
- CSV 和 Excel 的日期字段会做基础标准化，但复杂格式仍建议用户提前整理
- 批量创建目前按顺序逐行执行，失败不会中断全量导入

## 9. 后续建议

建议后续按下面顺序演进：

1. 支持企业微信附件 `media_id` 下载
2. 增加重复任务检测与跳过策略
3. 增加导入前预校验和二次确认
4. 支持更多表头别名和字段映射配置
5. 将该能力正式纳入 `intent-routing.yaml` 或独立导入模块

## 10. 验收标准

满足以下条件可视为本地第一版可用：

- 能成功新增需求文档并纳入仓库
- 能通过 `npm run build` 构建通过
- 能通过 `--dry-run` 成功解析示例模板
- 企微回调能识别“导入任务”并在缺参时给出提示
- 不修改服务器文件或服务器配置
