## 3. 当前项目定位

`openclaw-zentao-pack` 已不是早期的精简版 skill。

当前它已经是一个围绕禅道企业版 11.5 工作流扩展的全流程技能包，目标是把以下能力统一到一个包中：

- 产品与模块管理
- 需求管理
- 项目与执行管理
- 任务管理
- 测试用例与测试单管理
- Bug 管理
- 发布与上线验收
- 企业微信消息回调
- 企业微信通讯录同步
- 企业微信用户同步到禅道
- 统一输出格式与模块化技能说明

## 4. 当前代码结构

当前技能包主要分为以下几层：

- `scripts/shared/`
  - 共享客户端与公共逻辑
  - 关键文件：`zentao_client.ts`、`wecom_client.ts`
- `scripts/queries/`
  - 所有查询型脚本
- `scripts/actions/`
  - 所有创建、更新、关联、状态流转型脚本
- `scripts/callbacks/`
  - 企业微信回调入口
- `scripts/replies/`
  - 企业微信消息回复封装
- `modules/`
  - 按业务流程拆分的模块说明
- `docs/overview/`
  - 总体说明、模块映射、权限和状态文档
- `references/`
  - 各业务流程参考文档

说明：

- 动作类入口现已统一收敛到 `scripts/actions/`。
- 根级 `scripts/*.ts` 仅保留少量非重复入口脚本：`create_product.ts`、`create_product_modules.ts`、`create_product_with_modules.ts`、`get_my_tasks.ts`、`sync_user.ts`。
- 当前结构已经稳定为 `shared / queries / actions / callbacks / replies / modules`。

- 避免误回退到主 `openclaw.json`

## 9. 当前企业微信用户信息现状

已经只读验证过：

- 可以查到当前组织内用户清单
- 可以查到 `userid`、姓名、部门、主部门、职务、`status`、`enable`
- 当前 API 原始返回中，`gender`、`mobile`、`email` 等字段仍为空

注意：

- 这不是展示层遗漏，而是当前服务器所用企微接口响应本身未返回这些字段
- 即便分别测试 `corp_secret` 和 `contact_secret`，结果也相同

因此后续做用户同步时，当前默认只能依赖基础字段。

为了后续清理目录和减少重复抽象，当前建议把模块按下面 4 组理解：
