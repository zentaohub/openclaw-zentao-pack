# openclaw-zentao-pack 技能包总体功能说明

本文档用于从整体视角说明 `openclaw-zentao-pack` 的能力边界、模块清单、脚本分层和典型使用场景，方便后续维护、扩展和权限治理。

## 1. 技能包定位

`openclaw-zentao-pack` 是一个围绕禅道业务流程构建的 OpenClaw 技能包，目标是把禅道、企业微信和 OpenClaw 主会话串成一套可执行、可查询、可同步、可编排的工作流能力。

它不是单一功能脚本集合，而是一套覆盖以下对象的流程化技能包：

- 产品
- 模块
- 需求
- 项目
- 执行 / 迭代
- 任务
- 测试用例
- 测试单
- Bug
- 发布
- 上线验收
- 企业微信用户与消息回调

## 2. 总体能力分层

从整体设计看，当前技能包可以分为 6 层能力。

### 2.1 基础连接层

负责与禅道、企业微信建立连接，提供共享客户端能力。

核心脚本：

- `scripts/shared/zentao_client.ts`
- `scripts/shared/wecom_client.ts`
- `scripts/zentao_client.ts`
- `scripts/wecom_client.ts`

主要职责：

- 禅道登录与会话缓存

## 4. 当前技能包特点

### 4.1 优点

- 覆盖禅道核心研发流程较完整
- 查询、动作、同步三类能力比较清晰
- 已按模块化方式拆分，适合后续继续扩展
- 已接入企业微信场景，适合机器人交互
- 文档、模块、脚本之间已经形成基本对应关系

### 4.2 当前不足

- 还缺少统一的权限控制层
- 角色与动作的绑定规则还没有在技能包内部统一落地
- 一些脚本入口仍然是直接执行，缺少统一鉴权中间层
- 文档虽然已转中文，但总体说明此前不够集中，现在通过本文档补齐

## 5. 后续建议

### 5.1 适合马上推进的方向

1. 基于本文档继续梳理“模块 -> 脚本 -> action 权限点”的映射表
2. 为技能包增加统一权限模型，支持按禅道组织角色限制功能
3. 给企业微信机器人增加角色识别与指令白名单
4. 继续补齐每个模块的典型输入、典型输出和适用角色说明

### 5.2 建议新增的配套文档

- 《技能包权限模型说明》
- 《模块与脚本映射清单》
- 《角色与功能边界说明》
- 《企业微信机器人指令路由说明》

## 6. 文档规则

- 本技能包的总体说明、模块说明、参考说明统一使用中文。
- 后续新增说明文档也必须使用中文。
- 命令、脚本名、字段名、接口路径保留英文原文即可。
