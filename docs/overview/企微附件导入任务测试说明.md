# 企微附件导入任务测试说明

本文档用于说明如何在不修改服务器代码的前提下，验证“企微附件上传 + 禅道批量导入任务”链路。

## 1. 当前结论

当前本地代码已经兼容两种入口：

- URL 导入
- 企微附件导入

本地附件导入已进入真实 `media.get` 接口，但被企微返回：

```text
60020 not allow to access from your ip
```

这说明：

- 代码链路已经接通
- 当前阻塞点是企微 IP 白名单，而不是功能未实现

## 2. 推荐验证方式

优先在服务器上做附件导入测试，因为服务器更可能已经具备企微允许访问条件。

当前服务器技能目录通常为：

```text
/root/.openclaw/workspace/skills/openclaw-zentao-pack
```

## 3. 测试前提

需要准备：

- 一个真实的企微文件消息回调 JSON
- 其中包含：
  - `userid`
  - `media_id`
  - `fileName`
  - 文本中或内容中带上 `执行 4` 这类执行 ID

建议 payload 结构类似：

```json
{
  "userid": "admin",
  "msgtype": "file",
  "content": "导入任务 执行 4",
  "body": {
    "media_id": "真实的企微media_id",
    "fileName": "task-import-template.xlsx"
  }
}
```

## 4. 服务器测试命令

进入服务器技能目录后执行：

```bash
npm run build
node dist/scripts/callbacks/wecom_callback.js --data-file examples/callbacks/tmp-callback-import-file.json
```

如果不想落文件，也可以直接内联：

```bash
node dist/scripts/callbacks/wecom_callback.js --data '{"userid":"admin","msgtype":"file","content":"导入任务 执行 4","body":{"media_id":"真实的企微media_id","fileName":"task-import-template.xlsx"}}'
```

## 5. 预期结果

如果服务器 IP 在企微允许范围内，并且 `media_id` 可用，则预期会出现以下两类结果之一：

### 5.1 成功导入

返回内容包含：

- `success_count`
- `skipped_count`
- `failed_count`
- `reply_text`

### 5.2 已存在同名任务被跳过

返回内容包含：

- `success_count: 0`
- `skipped_count > 0`
- `reply_text` 中明确写出“同一执行下已存在同名任务，已跳过”

## 6. 常见失败含义

### 6.1 `60020 not allow to access from your ip`

说明企微素材下载接口拒绝当前出口 IP。

### 6.2 `Missing WeCom credentials`

说明当前环境没有加载到企微配置。

### 6.3 `execution/执行ID` 缺失

说明附件已被识别，但消息里没带执行 ID。

### 6.4 下载成功但导入失败

通常是：

- Excel 表头不符合要求
- 执行 ID 不存在
- 禅道对象权限或关联对象异常

## 7. 本地与服务器分工建议

建议按下面方式使用：

1. 本地负责开发、构建、dry-run、URL 测试
2. 服务器负责真实附件下载验证
3. 如果服务器附件测试通过，再决定是否补更多体验优化
