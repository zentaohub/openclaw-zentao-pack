# 灰度上线验收清单

## 1. 使用说明

这份清单适用于：

- 开发自测
- 联调确认
- 试点灰度上线前检查
- 试点期间每日回顾

建议在试点期连续使用 3 到 5 个工作日。

## 2. 上线前准备

### 2.1 配置准备

- [ ] 已准备真实配置文件
- [ ] 已校验 `userid` 正确
- [ ] 已校验 `zentaoAccount` 正确
- [ ] 已确认角色填写合法
- [ ] 已确认管理角色 `scope` 有效
- [ ] 已确认时区为 `Asia/Shanghai`
- [ ] 已确认 morning / evening cron 正确

### 2.2 环境准备

- [ ] 企业微信自建应用可正常主动发消息
- [ ] 自建应用对目标成员可见
- [ ] 禅道查询脚本在目标环境可正常运行
- [ ] 审计日志路径可写
- [ ] 定时任务执行环境已加载正确配置

## 3. 功能验收

### 3.1 早报

- [ ] 09:00 可以稳定触发
- [ ] 每个试点用户只收到 1 条摘要
- [ ] 摘要长度可在一屏内读完
- [ ] 总览数字与实际数据大体一致
- [ ] 重点项不超过 3 条
- [ ] 有可用详情入口

### 3.2 晚报

- [ ] 18:00 可以稳定触发
- [ ] 晚报内容与“今日收口”定位一致
- [ ] 不重复展开白天已讲过的普通明细
- [ ] 未收口风险能被正确突出

### 3.3 多角色合并

- [ ] 多角色用户仍只收到 1 条消息
- [ ] 消息最多 2 个分区
- [ ] 风险优先于普通待办

## 4. 风险准确性验收

### 4.1 项目 / 执行延期

- [ ] 已延期项目能进入管理摘要
- [ ] 首次延期时可进入即时提醒候选
- [ ] 未延期项目不会误报

### 4.2 任务延期

- [ ] 超期任务能进入研发摘要
- [ ] blocked 且超期任务会升级
- [ ] 已完成任务不会继续被当成延期

### 4.3 Bug 超期

- [ ] 高优 Bug 超过阈值会进入重点
- [ ] 普通 Bug 不会抢占高优风险位置
- [ ] 已关闭 Bug 不再继续提醒

### 4.4 测试准出

- [ ] failed / blocked / unrun / unresolved bug 能正确进风险
- [ ] 测试与管理收到的视角符合角色定位

## 5. 用户体验验收

- [ ] 用户反馈“消息够短”
- [ ] 用户知道先处理哪件事
- [ ] 用户不会觉得被频繁打扰
- [ ] 标题和文案没有明显歧义
- [ ] 不存在大段无效说明

## 6. 稳定性验收

- [ ] 单个用户发送失败不影响其他用户
- [ ] 失败用户可重试
- [ ] 同一时间槽重跑不会重复发
- [ ] 有发送审计记录
- [ ] 能查到失败原因

## 7. 试点反馈记录建议

每天至少收集这几类反馈：

- 哪条消息太长
- 哪类风险误报
- 哪类风险漏报
- 哪个入口不够清楚
- 是否需要调整阈值

## 8. 放量条件

满足以下条件再建议扩大范围：

- 连续 3 到 5 个工作日稳定发送
- 没有严重误报
- 没有大量重复提醒
- 试点用户普遍认为消息可读、可用
- 配置维护流程已明确
